## What is Strassen Grokking?

Strassen Grokking is a research project that demonstrates how neural networks can be induced to learn discrete algorithmic structures through controlled training conditions. Specifically, it shows that by carefully managing batch size, training duration, and regularization, a neural network can crystallize into the exact Strassen matrix multiplication algorithm.

<Note>
This is not algorithm *discovery* - it's algorithm *induction* through precise engineering. The network learns to represent a known structure (Strassen's 7-product formula) using integer coefficients in {-1, 0, 1}.
</Note>

## The Core Problem

Matrix multiplication normally requires 8 multiplications for 2×2 matrices. Strassen's algorithm achieves the same result with only 7 multiplications:

```python
C = A @ B  # Standard approach: 8 products
C = Strassen(A, B)  # Strassen's approach: 7 products
```

The challenge: **Can a neural network learn this exact discrete structure and generalize it to larger matrices without retraining?**

## Key Findings

### Success Rate

Under optimal conditions, **68% of training runs** (133 out of 195) successfully crystallize into a verifiable Strassen structure. The remaining 32% converge to local minima that generalize but don't pass structural verification.

<Warning>
Without the two-phase protocol (training + sparsification), 0% of runs achieve the Strassen structure. Explicit sparsification is essential.
</Warning>

### The Two-Phase Protocol

The method consists of two critical phases:

<Steps>
  <Step title="Phase 1: Thermodynamic Grokking">
    Train a bilinear model with 8 slots on 2×2 matrix multiplication using:
    - Batch sizes in [24, 128]
    - Weight decay ≥ 1e-4
    - Training for 1000+ epochs until grokking occurs
  </Step>
  
  <Step title="Phase 2: Sparsification and Discretization">
    - Prune to exactly 7 active slots based on importance scores
    - Round all weights to integers in {-1, 0, 1}
    - Verify the structure transfers to 64×64 matrices
  </Step>
</Steps>

## Extreme Fragility

The algorithmic structure exists in a narrow basin of weight space:

- Adding Gaussian noise σ ≥ 0.001 to trained weights causes **100% discretization failure**
- This fragility is not a bug - it's the fundamental reason why precise engineering is essential
- Small perturbations completely destroy the discrete attractor

<Note>
This fragility has implications beyond Strassen. If a well-defined algorithm requires such precise conditions to emerge, what does this say about reproducibility in deep learning more broadly?
</Note>
