## Overview

The benchmark module tests the Strassen algorithm across multiple matrix resolutions using the same grokked 2×2 operator recursively. It validates correctness, measures performance, and demonstrates zero-shot scaling to arbitrary power-of-2 dimensions.

## Import

```python
from benchmark_strassen import (
    BenchmarkConfig,
    BenchmarkResult,
    load_config,
    run_benchmark,
    benchmark_resolution,
    save_results,
    estimate_memory_mb,
    main
)
```

## BenchmarkConfig

```python
@dataclass
class BenchmarkConfig
```

Configuration for benchmark execution. Can be loaded from `config.toml` or created programmatically.

### Attributes

<ParamField path="resolutions" type="List[int]" default="[]">
  List of matrix dimensions to test (must be powers of 2). Example: `[2, 4, 8, 16, 32, 64, 128]`
</ParamField>

<ParamField path="max_memory_gb" type="float" default="8.0">
  Maximum memory budget in GB. Resolutions exceeding this will be skipped.
</ParamField>

<ParamField path="dtype" type="str" default="float32">
  Data type for matrices. Options: `"float32"`, `"float64"`, `"int32"`, `"int64"`
</ParamField>

<ParamField path="warmup_iterations" type="int" default="3">
  Number of warmup runs before timing (to warm caches)
</ParamField>

<ParamField path="test_iterations" type="int" default="10">
  Number of timed iterations to average
</ParamField>

<ParamField path="samples_per_resolution" type="int" default="100">
  Number of random test matrices per resolution (for correctness validation)
</ParamField>

<ParamField path="tolerance_float" type="float" default="1e-4">
  Error tolerance for floating-point tests
</ParamField>

<ParamField path="tolerance_int" type="float" default="0">
  Error tolerance for integer tests (exact match required)
</ParamField>

<ParamField path="verbose" type="bool" default="true">
  Print detailed progress information
</ParamField>

<ParamField path="save_results" type="bool" default="true">
  Save benchmark results to JSON file
</ParamField>

<ParamField path="results_file" type="str" default="benchmark_results.json">
  Path to results file
</ParamField>

<ParamField path="verify_correctness" type="bool" default="true">
  Run correctness verification against standard multiplication
</ParamField>

<ParamField path="compare_with_standard" type="bool" default="true">
  Time standard multiplication for speedup comparison
</ParamField>

### Example

```python
from benchmark_strassen import BenchmarkConfig

# Create config programmatically
config = BenchmarkConfig(
    resolutions=[2, 4, 8, 16, 32],
    dtype="float32",
    test_iterations=20,
    max_memory_gb=4.0
)

# Or load from TOML
config = load_config("config.toml")
```

## BenchmarkResult

```python
@dataclass
class BenchmarkResult
```

Results for a single resolution benchmark.

### Attributes

<ResponseField name="resolution" type="int">
  Matrix dimension (n for n×n matrix)
</ResponseField>

<ResponseField name="strassen_time_ms" type="float">
  Average Strassen execution time in milliseconds
</ResponseField>

<ResponseField name="standard_time_ms" type="float">
  Average standard multiplication time in milliseconds
</ResponseField>

<ResponseField name="speedup" type="float">
  Speedup factor: `standard_time / strassen_time`
</ResponseField>

<ResponseField name="strassen_mults" type="int">
  Number of scalar multiplications in Strassen algorithm: 7^(log₂ n)
</ResponseField>

<ResponseField name="standard_mults" type="int">
  Number of scalar multiplications in standard algorithm: n³
</ResponseField>

<ResponseField name="mult_reduction" type="float">
  Multiplication reduction ratio: `1 - (strassen_mults / standard_mults)`
</ResponseField>

<ResponseField name="max_error" type="float">
  Maximum absolute error vs standard multiplication
</ResponseField>

<ResponseField name="mean_error" type="float">
  Mean absolute error vs standard multiplication
</ResponseField>

<ResponseField name="memory_mb" type="float">
  Estimated memory usage in megabytes
</ResponseField>

<ResponseField name="passed" type="bool">
  Whether correctness test passed (error within tolerance)
</ResponseField>

<ResponseField name="error_msg" type="str">
  Error message if benchmark failed (empty string if passed)
</ResponseField>

### Example

```python
from benchmark_strassen import BenchmarkResult

# Results are typically generated by run_benchmark()
result = BenchmarkResult(
    resolution=16,
    strassen_time_ms=0.234,
    standard_time_ms=0.312,
    speedup=1.33,
    strassen_mults=2401,
    standard_mults=4096,
    mult_reduction=0.414,
    max_error=1.23e-6,
    mean_error=3.45e-8,
    memory_mb=2.5,
    passed=True
)

print(f"{result.resolution}x{result.resolution}: {result.speedup:.2f}x speedup")
```

## estimate_memory_mb

```python
def estimate_memory_mb(n: int, dtype: torch.dtype, batch_size: int = 1) -> float
```

Estimates memory usage for matrix multiplication at given resolution.

### Parameters

<ParamField path="n" type="int" required>
  Matrix dimension
</ParamField>

<ParamField path="dtype" type="torch.dtype" required>
  Data type (e.g., `torch.float32`)
</ParamField>

<ParamField path="batch_size" type="int" default="1">
  Number of matrices in batch
</ParamField>

### Returns

<ResponseField name="memory_mb" type="float">
  Estimated memory usage in megabytes, including:
  - Storage for A, B, C matrices
  - Intermediate results during recursion
  - Recursion stack overhead
</ResponseField>

### Example

```python
import torch
from benchmark_strassen import estimate_memory_mb

# Estimate for different resolutions
for n in [16, 32, 64, 128]:
    mem_fp32 = estimate_memory_mb(n, torch.float32)
    mem_fp64 = estimate_memory_mb(n, torch.float64)
    print(f"{n:>3}x{n:<3}: {mem_fp32:.1f} MB (fp32), {mem_fp64:.1f} MB (fp64)")

# Check if resolution fits in memory budget
max_mem_gb = 4.0
n = 256
mem_mb = estimate_memory_mb(n, torch.float32)
if mem_mb < max_mem_gb * 1024:
    print(f"{n}x{n} fits in {max_mem_gb} GB budget")
else:
    print(f"{n}x{n} exceeds {max_mem_gb} GB budget")
```

## benchmark_resolution

```python
def benchmark_resolution(n: int, cfg: BenchmarkConfig, dtype: torch.dtype) -> BenchmarkResult
```

Benchmarks Strassen vs standard multiplication for a single matrix dimension.

### Parameters

<ParamField path="n" type="int" required>
  Matrix dimension (must be power of 2)
</ParamField>

<ParamField path="cfg" type="BenchmarkConfig" required>
  Benchmark configuration
</ParamField>

<ParamField path="dtype" type="torch.dtype" required>
  Data type for matrices
</ParamField>

### Returns

<ResponseField name="result" type="BenchmarkResult">
  Complete benchmark results including timing, accuracy, and metadata
</ResponseField>

### Benchmark Process

1. **Generate test matrices** (random normal or random integers)
2. **Warmup phase** (cfg.warmup_iterations runs)
3. **Time Strassen** (average over cfg.test_iterations)
4. **Time standard** (average over cfg.test_iterations)
5. **Verify correctness** (compare outputs element-wise)
6. **Calculate statistics** (speedup, multiplication counts, etc.)

### Example

```python
import torch
from benchmark_strassen import benchmark_resolution, BenchmarkConfig

config = BenchmarkConfig(
    test_iterations=20,
    warmup_iterations=5,
    tolerance_float=1e-4
)

result = benchmark_resolution(32, config, torch.float32)

print(f"Resolution: {result.resolution}x{result.resolution}")
print(f"Strassen: {result.strassen_time_ms:.3f} ms")
print(f"Standard: {result.standard_time_ms:.3f} ms")
print(f"Speedup: {result.speedup:.2f}x")
print(f"Max error: {result.max_error:.2e}")
print(f"Passed: {result.passed}")
```

## run_benchmark

```python
def run_benchmark(cfg: BenchmarkConfig) -> List[BenchmarkResult]
```

Runs complete benchmark suite across all configured resolutions.

### Parameters

<ParamField path="cfg" type="BenchmarkConfig" required>
  Benchmark configuration with resolutions list
</ParamField>

### Returns

<ResponseField name="results" type="List[BenchmarkResult]">
  List of results for each resolution, in order
</ResponseField>

### Output Format

Prints formatted table during execution:

```
======================================================================

======================================================================
Resolutions: [2, 4, 8, 16, 32, 64]
Data type: float32
Max memory: 8.0 GB
Iterations: 10
======================================================================

     N |   Strassen |   Standard |  Speedup | Mults Red |     MaxErr |  Mem MB |   Status
-----------------------------------------------------------------------------------------------
     2 |      0.012 |      0.008 |     0.67x |     12.5% |   1.19e-07 |      0.0 |     PASS
     4 |      0.034 |      0.025 |     0.74x |     23.4% |   3.45e-07 |      0.0 |     PASS
     8 |      0.089 |      0.112 |     1.26x |     33.0% |   8.23e-07 |      0.0 |     PASS
    16 |      0.267 |      0.523 |     1.96x |     41.4% |   2.34e-06 |      0.1 |     PASS
    32 |      1.123 |      3.456 |     3.08x |     48.7% |   5.67e-06 |      0.3 |     PASS
    64 |      5.234 |     21.567 |     4.12x |     55.1% |   1.23e-05 |      1.3 |     PASS

Maximum successful resolution: 64x64
Average multiplication reduction: 35.7%

```

### Example

```python
from benchmark_strassen import load_config, run_benchmark

# Load config and run
config = load_config("config.toml")
results = run_benchmark(config)

# Analyze results
passed = [r for r in results if r.passed]
print(f"\n{len(passed)}/{len(results)} tests passed")

if passed:
    best_speedup = max(r.speedup for r in passed)
    best_result = max(passed, key=lambda r: r.speedup)
    print(f"Best speedup: {best_speedup:.2f}x at {best_result.resolution}x{best_result.resolution}")
```

## Typical Benchmark Results

### Performance Characteristics

<Tabs>
  <Tab title="Small Matrices (n ≤ 16)">
    **Standard multiplication is faster**
    
    - Highly optimized BLAS routines
    - Overhead of recursion dominates
    - Speedup: 0.5x - 1.0x
    
    ```python
    # Example results
    #   2x2: 0.67x speedup (12.5% mult reduction)
    #   4x4: 0.74x speedup (23.4% mult reduction)
    #   8x8: 1.02x speedup (33.0% mult reduction)
    #  16x16: 1.35x speedup (41.4% mult reduction)
    ```
  </Tab>
  
  <Tab title="Medium Matrices (32 ≤ n ≤ 128)">
    **Strassen becomes competitive**
    
    - Multiplication savings offset overhead
    - Cache effects become important
    - Speedup: 1.5x - 3.0x
    
    ```python
    # Example results
    #  32x32: 2.15x speedup (48.7% mult reduction)
    #  64x64: 2.89x speedup (55.1% mult reduction)
    # 128x128: 3.21x speedup (60.7% mult reduction)
    ```
  </Tab>
  
  <Tab title="Large Matrices (n ≥ 256)">
    **Strassen dominates**
    
    - Multiplication count is bottleneck
    - Approaching theoretical O(n^2.807)
    - Speedup: 3.0x+
    
    ```python
    # Example results (when memory permits)
    # 256x256: 3.87x speedup (65.2% mult reduction)
    # 512x512: 4.23x speedup (69.0% mult reduction)
    ```
  </Tab>
</Tabs>

### Accuracy

<Note>
  **Numerical stability**: The grokked coefficients are discrete ({-1, 0, 1}), ensuring:
  - No floating-point drift in coefficients
  - Error scales linearly with matrix size: O(n · ε)
  - Typical max error: < 1e-5 for n ≤ 128
  - Exact results for integer matrices (after rounding)
</Note>

## Complete Benchmark Example

```python
import torch
from benchmark_strassen import (
    BenchmarkConfig,
    load_config,
    run_benchmark,
    save_results,
    estimate_memory_mb
)

# Option 1: Use config.toml
config = load_config("config.toml")
results = run_benchmark(config)
if config.save_results:
    save_results(results, config.results_file)

# Option 2: Programmatic configuration
config = BenchmarkConfig(
    resolutions=[2, 4, 8, 16, 32, 64],
    dtype="float32",
    test_iterations=20,
    warmup_iterations=5,
    max_memory_gb=4.0,
    tolerance_float=1e-4,
    verbose=True,
    compare_with_standard=True
)

# Check memory constraints
print("Memory estimates:")
for n in config.resolutions:
    mem = estimate_memory_mb(n, torch.float32)
    status = "OK" if mem < config.max_memory_gb * 1024 else "SKIP"
    print(f"  {n:>3}x{n:<3}: {mem:>6.1f} MB - {status}")

# Run benchmarks
print("\nRunning benchmarks...")
results = run_benchmark(config)

# Analyze results
passed = [r for r in results if r.passed]
failed = [r for r in results if not r.passed]

print(f"\n{'='*70}")
print("ANALYSIS")
print(f"{'='*70}")
print(f"Tests: {len(passed)} passed, {len(failed)} failed")

if passed:
    # Best speedup
    best = max(passed, key=lambda r: r.speedup)
    print(f"\nBest speedup: {best.speedup:.2f}x at {best.resolution}x{best.resolution}")
    
    # Average metrics
    avg_speedup = sum(r.speedup for r in passed) / len(passed)
    avg_reduction = sum(r.mult_reduction for r in passed) / len(passed)
    print(f"Average speedup: {avg_speedup:.2f}x")
    print(f"Average mult reduction: {avg_reduction*100:.1f}%")
    
    # Maximum resolution
    max_res = max(r.resolution for r in passed)
    print(f"Maximum resolution: {max_res}x{max_res}")
    
    # Accuracy summary
    max_error = max(r.max_error for r in passed)
    mean_error = sum(r.mean_error for r in passed) / len(passed)
    print(f"\nMax error: {max_error:.2e}")
    print(f"Mean error: {mean_error:.2e}")

# Save results
save_results(results, "my_benchmark.json")
print(f"\nResults saved to my_benchmark.json")
```
