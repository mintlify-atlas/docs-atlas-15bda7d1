This work demonstrates that neural networks can be engineered to learn the exact Strassen matrix multiplication algorithm through precise control of training conditions. The approach is empirical materials engineering rather than theoretical derivation.

## Two-Phase Protocol

Algorithmic structure induction requires a two-phase approach:

<Steps>
  <Step title="Phase 1: Thermodynamic Grokking">
    Train a bilinear model with 8 slots on 2×2 matrix multiplication using AdamW optimizer with controlled batch size and weight decay until grokking occurs.
    
    ```python
    model = StrassenOperator(rank=8)
    optimizer = optim.AdamW(model.parameters(), lr=0.02, weight_decay=1e-4)
    # Train until: training loss < 1e-6 and test accuracy reaches 100%
    ```
  </Step>
  
  <Step title="Phase 2: Sparsification and Discretization">
    After training, prune to exactly 7 active slots according to importance scores, then discretize all weights to integers in {-1, 0, 1} using rounding.
    
    ```python
    # Prune least important slot
    importance = model.slot_importance()  # L2 norm per slot
    # Prune to 7 slots, then round to {-1, 0, 1}
    ```
  </Step>
</Steps>

<Note>
  **Both phases are necessary.** Phase 1 alone achieves 0% success. The model converges to solutions with 8 active slots and non-integer weights that do not match the Strassen structure. Only after implementing Phase 2 with explicit sparsification does the protocol achieve 68% success rate.
</Note>

## Critical Training Conditions

The following conditions are required for successful discretization based on 245 training runs:

### Batch Size Range

**Optimal range: [24, 128]**

Batch size is the dominant control parameter. Statistical analysis across 195 runs shows:

- **F-statistic:** 15.34 (p < 0.0001)
- **Effect size (η²):** 0.244 (explains 24% of variance)
- **Optimal values:** No significant differences between B ∈ {24, 32, 64}

```text
Batch Size Effect on Success Rate:
  B=8:    Low success (high noise, unstable)
  B=24:   ✓ Optimal
  B=32:   ✓ Optimal 
  B=64:   ✓ Optimal
  B=96:   ✓ Optimal
  B=128:  ✓ Optimal
  B=256:  Decreased success
  B=512:  Poor (trapped in glass phase)
```

<Warning>
  The batch size range [24-128] creates a latent structure ready to crystallize. Smaller batch sizes (B=8) remain too unstable to align, while larger batch sizes (B=512) increase h̄ entropy, trapping the model in a dense glass that is much harder to distill into a pure algorithm.
</Warning>

### Training Duration

**Minimum: 1000 epochs with observed grokking**

Extended training is required for weights to approach near-integer values before discretization. Grokking is operationally defined as:

- Training loss < 10⁻⁶ while test loss > 0.1 for at least 100 epochs
- Followed by sudden drop in test loss
- Final test accuracy reaches 100%

Example training progression:

```text
500   | 33    | 3.96e-04  | 0.0   | 0.0034 | 3033.67   | Training

```

### Weight Decay

**Required: ≥ 1e-4 for AdamW**

Weight decay functions as thermodynamic pressure, helping weights collapse toward smaller magnitudes that are easier to discretize.

```python
optimizer = optim.AdamW(
    model.parameters(), 
    lr=0.02, 
    weight_decay=1e-4  # Critical threshold
)
```

<Note>
  AdamW with weight decay ≥1e-4 produces significantly better results than pure Adam. The regularization appears essential for reaching the discrete attractor basin.
</Note>

### Symmetric Initialization

**Required: U and V tensors must use symmetric initialization**

The bilinear model requires symmetric initialization for U and V tensors:

```python
self.U = nn.Parameter(torch.randn(rank, 4) * 0.5)
self.V = nn.Parameter(torch.randn(rank, 4) * 0.5)
```

## Success Metrics

### Definition of Success

A run achieves success if and only if:

1. **Discretization Success:** All 21 weight values (7 slots × 3 tensors) satisfy |w - round(w)| < 0.5 AND the rounded values match a valid Strassen coefficient structure

2. **Expansion Success:** The discretized coefficients pass verification at all scales (2×2, 4×4, 8×8, 16×16, 32×32, 64×64) with relative error < 1e-5

3. **No Fallback:** Success is counted only when using learned coefficients without any intervention

### Achieved Results

**Success rate: 68% (133/195 runs)**

The reported 68% refers to runs that achieved BOTH discretization success and expansion success using only learned coefficients. The remaining 32% of runs failed discretization or required falling back to canonical Strassen coefficients.

```text
Success Rate by Condition:
  With two-phase protocol:     68% (133/195)
  Without sparsification:       0% (0/195)
  With noise (σ ≥ 0.001):      0% (0/100)
```

## Order Parameters for Monitoring

Three metrics track the phase transition:

### κ (Kappa): Condition Number

The condition number of the gradient covariance matrix acts as temperature:

- **κ = 1.000:** Crystalline phase (algorithm learned)
- **κ > 10:** Glass phase (local minimum)
- **AUC: 1.000** (perfect separation, 95% CI [1.000, 1.000])

See [Kappa Metric](/metrics/kappa) for implementation details.

### δ (Delta): Discretization Margin

Measures distance from integer values:

```python
δ = max(|w - round(w)|) for all weights
```

- **δ = 0:** Perfect crystal (all weights at integer values)
- **δ < 0.1:** Within discretization margin
- **δ ≈ 0.5:** Amorphous glass

See [Delta Metric](/metrics/delta) for prospecting applications.

### Accuracy

Test accuracy must reach 100% and remain stable:

- **< 100%:** Still in training phase
- **= 100% sustained:** Crystallization candidate
- **Drops from 100%:** Phase transition failed

## Engineering Recipe

The working protocol for inducing Strassen structure:

<Steps>
  <Step title="Configure Training">
    - Batch size: 24-128 (recommend 32)
    - Weight decay: 1e-4 or higher
    - Learning rate: 0.02 with cosine annealing
    - Random seed: test multiple (seed prospecting with δ)
  </Step>
  
  <Step title="Train Until Crystallization">
    - Monitor κ, δ, and accuracy every 100 epochs
    - Continue until: κ ≈ 1, Acc = 100%, δ → 0
    - Minimum 1000 epochs, typical 5000-20000
    - Multiple crystallization events may occur
  </Step>
  
  <Step title="Apply Sparsification">
    - Compute slot importance: ||U[k]|| × ||V[k]|| × ||W[:,k]||
    - Prune least important slot
    - Refine for 10,000 epochs with pruned slot frozen
  </Step>
  
  <Step title="Discretize and Verify">
    - Round all weights to nearest integer
    - Clamp to {-1, 0, 1}
    - Verify at scales: 2×2, 4×4, 8×8, 16×16, 32×32, 64×64
    - Require max error < 1e-5 at all scales
  </Step>
</Steps>

<Warning>
  This is not algorithm discovery—it is engineering a known structure through strong priors and explicit intervention. What is novel is the engineering protocol that makes this induction reliable and verifiable.
</Warning>
