## Thermodynamic Interpretation

### Gradient Noise Variance

Batch size controls gradient noise through the relationship:

```python
Var(∇L) ∝ 1/B  # Gradient variance inversely proportional to batch size
```

The effective Planck constant quantifies gradient noise entropy:

```python
h̄_eff = 2.0 * δ² * λ  # From uncertainty relation
```

**Finding:** Optimal batch sizes maintain h̄_eff in the narrow window [9e-3, 1.6e-2], allowing exploration while enabling crystallization.

## The Goldilocks Principle

Batch size creates a three-regime system:

### Too Small (B < 24): High Noise Regime

**Problem:** Gradient variance too high for stable convergence

```text
Characteristics:
  - Var(∇L) > 0.01
  - h̄_eff > 0.1
  - Training path chaotic
  - Cannot align to narrow basin
  - Success rate: 12-16%
```

**Mechanism:** The discrete basin has width ~0.001 in weight space (from noise perturbation experiments). With gradient noise σ ≈ 0.18, the random walk step size exceeds basin width by 180×, making stable entry nearly impossible.

### Optimal (24 ≤ B ≤ 128): Goldilocks Zone

**Characteristics:**

```text
  - Var(∇L) ∈ [0.002, 0.01]
  - h̄_eff ∈ [0.009, 0.016]
  - Balanced exploration-exploitation
  - Can find and stabilize in basin
  - Success rate: 67-74%
```

**Mechanism:** Gradient noise is large enough to escape shallow local minima but small enough to stabilize once the crystalline basin is reached. The system can explore during early training (epochs 0-1000) then settle during grokking (epochs 1000-5000).

### Too Large (B > 128): Low Noise / High Entropy Trap

**Problem:** Insufficient exploration OR elevated entropy traps system in glass

```text
Characteristics:
  - Var(∇L) < 0.001
  - h̄_eff elevated (B=512: 0.145)
  - Deterministic trajectories
  - Gets stuck in first minimum found
  - Success rate: 31% (B=256), 8% (B=512)
```

**Mechanism (B=256):** Low gradient noise leads to premature convergence in shallow basins.

**Mechanism (B=512):** Large batch sizes increase h̄ entropy, trapping the model in a dense, overly complicated glass that is much harder to distill into pure algorithm. The model learns a valid local minimum but cannot reach the simple algorithmic attractor.

```text
      256  │ 18%    43%    47%    44%    31%

```

**Interpretation:** The effect of batch size is **amplified** by adequate weight decay. With wd < 1e-4, even optimal batch sizes achieve only 23-31% success. With wd ≥ 1e-4, optimal batch sizes achieve 67-74% success.

<Warning>
  Batch size and weight decay act synergistically. You cannot compensate for inadequate weight decay (< 1e-4) by adjusting batch size, nor vice versa. Both must be in optimal ranges.
</Warning>

## Optimal Batch Size Calculation

From thermodynamic principles, optimal batch size can be estimated:

```python
# Theoretical prediction
B_sat = h̄_eff / Δ_struct

# Where:
#   h̄_eff = measured effective Planck constant
#   Δ_struct = structural gap (discretization margin)

# Empirical values:
h̄_eff = 0.012  # From successful runs
Δ_struct = 0.001  # Basin width from noise experiments

B_sat = 0.012 / 0.001 = 12
```

**Interpretation:** The saturation batch size B_sat ≈ 12 marks the lower bound where gradient noise becomes manageable. Above this threshold, success rate begins to climb.

Practical optimal range accounts for computational efficiency and starts around 2× B_sat:

```text
B_opt,practical ∈ [2 × B_sat, 10 × B_sat]
              = [24, 120]
```

This matches the empirically observed optimal range of [24, 128].

## Practical Guidelines

<Steps>
  <Step title="Choose Batch Size">
    Select from optimal range [24, 128] based on:
    - **GPU memory:** Larger batches need more memory
    - **Training speed:** Larger batches train faster per epoch
    - **Success rate:** All values in range achieve ~70%
    
    **Recommendation:** Use B=64 unless constrained
  </Step>
  
  <Step title="Set Weight Decay">
    Must use wd ≥ 1e-4 for batch size effect to manifest:
    - **Standard:** wd = 1e-4
    - **Aggressive:** wd = 5e-4 to 1e-3
    - **Never:** wd < 1e-4 (success rate collapses)
  </Step>
  
  <Step title="Monitor Phase">
    Track order parameters:
    - **κ:** Should decrease toward 1 during grokking
    - **δ:** Should fall to 0 at crystallization
    - **Accuracy:** Should reach 100% and stabilize
    
    If no progress after 10,000 epochs, the seed is trapped
  </Step>
  
  <Step title="Prospect Seeds">
    Test multiple seeds (10-20) for 100 epochs each:
    - Measure δ at epoch 100
    - Select seeds with δ < 0.05
    - Continue training only selected seeds
    
    See [Delta Metric](/metrics/delta) for details
  </Step>
</Steps>
